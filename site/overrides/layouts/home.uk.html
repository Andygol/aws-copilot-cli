Ваш інструментарій для контейнеризованих застосунків в AWS

AWS Copilot — це інтерфейс командного рядка з відкритим вихідним кодом, який полегшує розробникам створення, випуск та використання готових до експлуатації контейнеризованих застосунків на AWS App Runner, Amazon ECS та AWS Fargate.

Вперед

Почніть з архітектури

Виконайте одну команду, щоб швидко почати роботу з контейнеризованим застосунком, використовуючи найкращі практики на AWS з Dockerfile.

Замість моделювання окремих ресурсів, Copilot пропонує загальні хмарні архітектури: Вебсервіс на основі запитів, Вебсервіс з балансуванням навантаження, Бекенд-сервіс, Сервіс виконання робочих навантажень, Заплановане завдання. Необхідна інфраструктура генерується з обраного шаблону. Зосередьте свій час на написанні бізнес-логіки замість підключення ресурсів AWS.

Безперервна доставка

Не потрібно турбуватися про збирання команд Copilot у скрипт для створення автоматизованого процесу випуску. Copilot надає команди для створення кількох середовищ розгортання, середовищ в окремих облікових записах та регіонах AWS, а також створення конвеєра AWS CodePipeline для створення образів контейнерів, розгортання сервісів та запуску автоматизованих тестів.

Операційна діяльність є частиною робочого процесу

Моделювання, забезпечення та розгортання сервісів — це лише частина життєвого циклу застосунку для розробника. Copilot також підтримує робочі процеси, повʼязані з усуненням несправностей та налагодженням, щоб допомогти, коли щось піде не так. Переглядайте свої журнали, отримайте доступ до оболонки працюючого контейнера, переглядайте стан своїх сервісів з комфортом у вашому терміналі.
